Multi-Tenant Platform API

JWT multi-tenant REST API · version 1

Checking API…

Base URL

All endpoints return JSON in a standard envelope: data, message, traceId.

Getting started

  • POST /api/v1/auth/login — obtain access token
  • POST /api/v1/auth/refresh — refresh tokens
  • GET /api/v1/users/current — profile (Bearer token)

Tenant users must include tenantSlug on login. SuperAdmin omits it.

API areas

  • GET /api/v1/health — service status
  • * /api/v1/auth/* — authentication
  • * /api/v1/users · /api/v1/roles
  • * /api/v1/tenants · /api/v1/products
  • * /api/v1/permissions · /api/v1/reports · /api/v1/files

Documentation

Interactive Swagger UI is available in Development only. Use an HTTP client (Postman, curl, etc.) against this server with a Bearer token for production testing.
